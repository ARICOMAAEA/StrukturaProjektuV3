Meta Framework V3 — Architecture Overview

v1.6.0 — How the project-folder generator actually works

Two-layer architecture

  • • Knowledge layer: OneDrive, .md only
  • • Execution layer: C:\DEV, runnable code
  • • Bidirectional bridge: ExecutionLayer.lnk + CONTEXT/ junction
  • • Rule: runnable artifacts ALWAYS in execution

Automation in .claude/

  • • 5 skills (user-invokable, session-scoped)
  • • 4 agents (long-running, focused tasks)
  • • 1 hook (PowerShell pre-clear gate)
  • • Plus 2 global skills for template back-port

Lifecycle protocol

  • • CLAUDE.md auto-loads at session start
  • • Every execution change updates README.md
  • • zvl_end-session closes a session cleanly
  • • session-guard.ps1 blocks /clear if open
